# Weld AI 2.0 / detection_tmx

Прототип платформы видеоаналитики для автоматизированного контроля качества на производстве подвижного состава.

Проект объединяет детекцию и сегментацию дефектов сварных швов на базе YOLOv8, генерацию объяснимых отчётов, демо-анализ изображений и проверку последовательности технологических операций.

> Проект создан как демонстрационный прототип для хакатона РЖД 2026 «Видеоаналитика ИИ на страже качества в производстве».

## Возможности

- детекция и сегментация дефектов сварных швов;
- классификация шести классов состояния сварки;
- визуализация найденных областей;
- расчёт итоговой оценки качества;
- выделение критических дефектов;
- генерация текстового и JSON-отчёта;
- правило-базированный анализ возможных причин дефектов;
- экспериментальная генерация отчётов с помощью локальной LLM;
- синтетическая генерация демонстрационных изображений;
- контроль последовательности производственных операций;
- упрощённый трекинг объектов между кадрами;
- экспорт обученной модели в ONNX;
- веб-интерфейс на Streamlit.

## Классы модели

Модель обучается для распознавания шести классов:

| ID | Класс |
|---:|---|
| 0 | Bad Welding |
| 1 | Crack |
| 2 | Excess Reinforcement |
| 3 | Good Welding |
| 4 | Porosity |
| 5 | Spatters |

В текущей логике критическими считаются классы `Bad Welding`, `Crack` и `Porosity`.

## Установка

### 1. Клонирование репозитория

```bash
git clone https://github.com/Klu-qiqi/detection_tmx.git
cd detection_tmx
```

### 2. Создание виртуального окружения

Linux/macOS:

```bash
python3 -m venv .venv
source .venv/bin/activate
```

Windows PowerShell:

```powershell
python -m venv .venv
.venv\Scripts\Activate.ps1
```

### 3. Установка основных зависимостей

Для запуска веб-интерфейса и базового анализа:

```bash
pip install streamlit opencv-python ultralytics numpy pandas pillow pyyaml matplotlib
```

Для генеративных компонентов:

```bash
pip install torch transformers accelerate sentencepiece
```

Полный список зависимостей находится в `requirements.txt`.

> `onnxruntime-gpu`, `tensorrt`, `paho-mqtt` и `asyncua` нужны только для соответствующих сценариев промышленного развёртывания. TensorRT обычно устанавливается отдельно с учётом версии CUDA и операционной системы.

## Быстрый запуск

### Streamlit-интерфейс

```bash
streamlit run app.py
```

После запуска откройте адрес, который Streamlit выведет в терминале. Обычно это:

```text
http://localhost:8501
```

В интерфейсе можно выбрать категорию контроля, загрузить изображение или воспользоваться встроенными демо-данными.

## Запуск консольного анализатора

```bash
python demo_analyzer.py
```

Скрипт:

1. ищет изображения в `test/images`;
2. при отсутствии изображений создаёт синтетический пример;
3. запускает модель;
4. рассчитывает оценку качества;
5. сохраняет текстовый отчёт;
6. сохраняет аннотированное изображение и диаграмму дефектов.

Ожидаемые результаты:

```text
demo_results/annotated.jpg
demo_results/defects_chart.png
weld_quality_report.txt
```

По умолчанию анализатор ожидает файл весов:

```text
best_weld_ai.pt
```

Путь можно изменить при создании инспектора:

```python
from demo_analyzer import WeldQualityInspector

inspector = WeldQualityInspector("path/to/model.pt")
report = inspector.analyze_image("path/to/image.jpg")
```

## Обучение модели

Конфигурация датасета задаётся в `data.yaml`.

Текущая конфигурация использует:

```yaml
train: ../train/images
val: ../valid/images
test: ../test/images
nc: 6
```

Перед запуском проверьте пути. Если `data.yaml` находится в корне репозитория вместе с каталогами `train`, `valid` и `test`, обычно следует использовать:

```yaml
train: train/images
val: valid/images
test: test/images
```

Запуск обучения:

```bash
python train.py
```

Основные параметры:

- модель: `yolov8n-seg.pt`;
- размер изображения: `640`;
- число эпох: `50`;
- batch size: `32`;
- устройство: `device=0`;
- ранняя остановка: `patience=15`;
- аугментации: HSV, масштабирование, сдвиг, mosaic, mixup, copy-paste и random erasing.

После обучения скрипт пытается:

- выполнить валидацию;
- вывести mAP, precision и recall;
- сохранить веса;
- экспортировать модель в ONNX;
- сформировать `training_report.txt`.

### Запуск на CPU

В `train.py` жёстко задано:

```python
device=0
```

Для запуска без CUDA замените значение на:

```python
device="cpu"
```

При этом обучение будет значительно медленнее.

## Генеративный анализ

Файл `generative_ai.py` содержит класс `GenerativeDefectAnalyzer`.

Он реализует три экспериментальных сценария:

### Генерация идеального изделия

Метод `generate_ideal_product()` создаёт параметрическое эталонное изображение сварного шва или окрашенной поверхности.

В текущей версии это синтетическая OpenCV-генерация, а не полноценная diffusion-модель.

### Объяснимый отчёт

Метод `generate_explainable_report()` может использовать модель:

```text
TinyLlama/TinyLlama-1.1B-Chat-v1.0
```

Если LLM отключена или недоступна, применяется правило-базированный анализ.

По умолчанию singleton создаётся с параметром:

```python
enable_llm=False
```

Поэтому загрузка TinyLlama при стандартном сценарии не выполняется.

### Аугментация редких дефектов

Метод `augment_rare_defects()` синтетически добавляет поры или трещины на изображение.

Несмотря на упоминание GAN в документации класса, текущая реализация использует OpenCV и случайно сгенерированные примитивы.

## Контроль технологического процесса

`process_control.py` содержит:

- `ProcessValidator` — конечный автомат для проверки порядка операций;
- `ObjectTracker` — упрощённое сопоставление объектов по IoU.

Ожидаемая последовательность:

```text
Входной контроль
→ Сварка
→ Контроль сварки
→ Покраска
→ Финальная сборка
→ Финальный контроль качества
```

Валидатор обнаруживает:

- возврат к предыдущему этапу;
- пропуск обязательного этапа;
- критические нарушения последовательности.

Упоминание ByteTrack относится к планируемой промышленной реализации. В текущем коде используется собственное упрощённое сопоставление рамок по IoU.

## Архитектура

Общая логика решения:

```text
Изображение / видео / камера
          │
          ▼
  Предобработка OpenCV
          │
          ▼
 YOLOv8-seg / детектор аномалий
          │
          ├── визуализация дефектов
          ├── оценка критичности
          ├── расчёт метрик качества
          └── генерация объяснимого отчёта
                          │
                          ▼
                 Streamlit / JSON / MES
```

Целевая промышленная архитектура также предусматривает:

- подключение IP-камер;
- обработку RTSP-потоков;
- очереди сообщений;
- горизонтальное масштабирование;
- REST-интеграцию с MES/ERP;
- OPC UA для взаимодействия с оборудованием;
- MQTT;
- ONNX Runtime и TensorRT;
- развёртывание на edge-устройствах.

Большая часть этих интеграций описана как целевая архитектура и пока не реализована в рабочем виде.

## Текущие ограничения

Репозиторий находится в состоянии прототипа. Перед полноценным запуском необходимо учесть следующие моменты.

### Отсутствующие модули

`app.py` импортирует:

```python
from weld_detector import WeldDefectDetector
from anomaly_detector import AnomalyDetector
from generative_ai import GenerativeReportGenerator
```

Однако в текущем корне репозитория отсутствуют:

```text
weld_detector.py
anomaly_detector.py
```

Кроме того, в `generative_ai.py` определён класс `GenerativeDefectAnalyzer`, но класс `GenerativeReportGenerator`, импортируемый в `app.py`, не найден.

Благодаря обработке исключений интерфейс может перейти в демо-режим, однако полноценный инференс через `app.py` без этих компонентов недоступен.

### Несогласованные пути к весам

Разные части проекта ожидают разные имена и пути:

```text
app.py:             models/best_weld_ai.pt
demo_analyzer.py:   best_weld_ai.pt
train.py:           best_weld_ai.pt
```

Рекомендуется выбрать единый каталог, например:

```text
models/best_weld_ai.pt
```

и использовать его во всех файлах.

### Сохранение модели после обучения

В Ultralytics лучшие веса обычно сохраняются внутри каталога запуска, например:

```text
weld_ai_runs/production_demo/weights/best.pt
```

Вызов:

```python
model.save("best_weld_ai.pt")
```

может сохранять текущее состояние модели, а не автоматически выбирать лучший checkpoint. Надёжнее явно копировать файл `weights/best.pt`.

### Пути в `data.yaml`

Текущие пути начинаются с `../`, хотя папки датасета находятся в репозитории. Их необходимо проверить относительно расположения `data.yaml`.

### Расчёт доли дефектов

В `demo_analyzer.py` переменная `total_weld_area` увеличивается только для класса `Good Welding`. Из-за этого формула `defect_ratio` не отражает отношение площади дефектов ко всей площади шва.

Для корректного расчёта следует суммировать площади всех областей сварного шва и отдельно площади дефектных классов.

### LLM-ответ не разбирается

Метод `_parse_llm_response()` пока игнорирует текст, созданный моделью, и возвращает правило-базированный отчёт.

### Видеообработка

Интерфейс принимает видеофайлы для отдельных категорий, но в текущей реализации вместо обработки видео используется демо-заглушка.

### Заявленные промышленные интеграции

RTSP, ONVIF, MES, OPC UA, MQTT, RabbitMQ, Grafana, React, ByteTrack, полноценные diffusion-модели и промышленный LLM-пайплайн пока представлены главным образом в архитектурном описании.

## Пример программного использования


## Возможные проблемы

### `ModuleNotFoundError: No module named 'weld_detector'`

Модуль отсутствует в текущей версии репозитория. Необходимо добавить его реализацию или изменить `app.py`, чтобы использовать `WeldQualityInspector` либо напрямую `ultralytics.YOLO`.

### `ModuleNotFoundError: No module named 'anomaly_detector'`

Модуль отсутствует. При текущей обработке исключения приложение должно продолжить работу в демо-режиме, но реальный анализ несварочных категорий выполняться не будет.

### `ImportError: cannot import name 'GenerativeReportGenerator'`

В `generative_ai.py` нет такого класса. Следует изменить импорт и вызовы либо реализовать совместимый адаптер.

### `FileNotFoundError: data.yaml`

Запускайте `train.py` из корня репозитория либо передайте правильный путь к конфигурации.

### CUDA недоступна

Измените `device=0` на `device="cpu"` или укажите доступный GPU.

### Модель не найдена

Проверьте наличие файла весов и приведите путь к единому формату во всех скриптах.
